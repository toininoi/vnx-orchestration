# Frontend Developer Report Template

## Output Location
Write your report to: `.vnx-data/unified_reports/`

## Filename Format
`{timestamp}-{track}-frontend-{title}.md`

Example: `20260202-143000-A-frontend-dashboard-component.md`

## Report Structure

### Summary
Brief overview of frontend work completed (2-3 sentences)

### Tags (Required)
- [tag1, tag2, tag3]  # Use specific, compound tags (e.g., sse-streaming, browser-pool, kvk-validation). Avoid general-only tags.

### Implementation
- Components created/modified
- UI framework used (React/Vue/Angular)
- State management approach
- Styling methodology

### User Experience
- UI/UX improvements made
- Accessibility features added (ARIA, keyboard nav)
- Responsive design breakpoints
- Performance optimizations

### Code Changes
- Component files created/modified
- State management updates
- Routing changes
- API integration points

### Testing
- Component tests written
- E2E test scenarios
- Visual regression tests
- Accessibility audit results

### Evidence
- Screenshots of UI changes
- Component code snippets
- Performance metrics (Lighthouse scores)
- Browser compatibility results

### Recommendations
- Further UX improvements
- Performance optimizations
- Component refactoring needs
- Accessibility enhancements
