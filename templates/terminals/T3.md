# T3 Worker Terminal (Track C)

Purpose: deep analysis and higher-cognition investigations.

## Startup
1. Your dispatch arrives in this conversation (sent by the dispatcher). It contains PR-ID, Track, Gate, and the full instruction. Do NOT look in `.vnx-data/dispatches/` — that directory is managed by the system.
2. Perform deep analysis and produce recommendations based on the dispatch you received.
3. Write status/update report to `.vnx-data/unified_reports/`.

## Open Items in Reports (MANDATORY)

Your report MUST always include an `## Open Items` section — even when empty. The receipt processor parses this section to automatically create tracked deliverables for T0.

**When to add open items**: Any issue you discover during work that is NOT part of the current dispatch scope — technical debt, bugs in adjacent code, performance concerns, missing tests, security issues.

```markdown
## Open Items
- [ ] [blocker] Critical issue that blocks further work
- [ ] [warn] Issue that should be addressed soon
- [ ] [info] Improvement suggestion or minor concern
```

If no issues: `None - all work completed and tested.`
The system auto-generates OI-IDs — do NOT write IDs yourself.

## Guardrails
- Prioritize clear findings and risk assessment.
- Distinguish facts from assumptions in reports.
- Treat `.vnx-data/` as runtime state (do not commit).
